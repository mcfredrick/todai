## Model Releases
**[GLM-OCR Multilingual Optical Character Recognition Model](https://huggingface.co/zai-org/GLM-OCR)** — A vision‑language model built on the GLM architecture, fine‑tuned for OCR across Chinese, English, French, Spanish, Russian, German, Japanese, and Korean. MIT licensed, Safetensors/Transformers ready, with arxiv paper 2603.10.

## Open Source Releases
- **[chandra](https://github.com/datalab-to/chandra)** — Layout‑aware OCR that preserves tables, forms, and handwritten text while maintaining full document layout for higher accuracy on dense pages.
- **[LangChain-MinerU 0.1.3 – PDF‑to‑Markdown Loader for LangChain](https://pypi.org/project/langchain-mineru/0.1.3/)** — Loader that runs MinerU on PDFs, outputting structured Markdown with headings, tables, and code blocks; ideal for feeding clean text into RAG pipelines.
- **[Jupyter AI Extension 1.0.0 – Multi‑LLM Support for Notebooks](https://pypi.org/project/jupyter-ai-extension/1.0.0/)** — Sidebar lets you switch between OpenAI GPT, Claude, Gemini inside a notebook, triggering code generation, explanation, or debugging via natural‑language prompts.
- **[ty](https://github.com/astral-sh/ty)** — Rust‑based Python type checker and language server delivering near‑instant feedback on type errors, boosting productivity in large ML codebases.
- **[TaxHacker](https://github.com/vas3k/TaxHacker)** — Self‑hosted AI accounting app that uses LLMs to parse receipts, invoices, and transactions; keeps data local and extensible via plugins.
- **[Aye Chat: Terminal-first AI Code Generator](https://pypi.org/project/ayechat-dev/0.45.1.20260320111503/)** — Terminal UI for prompting AI models to emit code snippets, integrating directly with local development workflows to speed up coding.
- **[IntentKit 0.13.0 – Core Package for Intent-based AI Agents](https://pypi.org/project/intentkit/0.13.0/)** — Framework for building agents around user intents, providing recognition, dialogue, action modules and improved async support.

## Research Worth Reading
- **[Efficient Dense Crowd Trajectory Prediction Via Dynamic Clustering](https://arxiv.org/abs/2603.18166)** — Introduces a dynamic clustering method to predict trajectories in occluded crowds, improving scalability and accuracy for safety‑critical applications like stampede prevention.

## AI Dev Tools
- **[vllm-omni](https://github.com/vllm-project/vllm-omni)** — Extension of vLLM for efficient omni‑modality inference (vision‑language, audio‑language) with optimized memory usage and batching.
- **[adk-python](https://github.com/google/adk-python)** — Google’s Agent Development Kit in Python; offers modular planning, tool use, and memory components compatible with various LLM backends.
- **[huggingface/skills](https://github.com/huggingface/skills)** — Toolkit giving agents abilities to load, infer, and fine‑tune Hugging Face models; includes abstractions for prompt chaining, RAG, and tool use.
- **[apm](https://github.com/microsoft/apm)** — Microsoft’s Agent Package Manager for discovering, installing, and versioning AI agent packages across teams.
- **[anthropics/skills](https://github.com/anthropics/skills)** — Community‑driven skills repository for Claude agents (web search, code execution, data analysis) encouraging reuse and sharing.
- **[warp](https://github.com/NVIDIA/warp)** — GPU‑kernel framework for accelerated simulation, data generation, and spatial computing; enables differentiable physics for ML research.
- **[opencode](https://github.com/anomalyco/opencode)** — Open‑source coding agent that assists with generation, debugging, and refactoring via natural language, IDE‑agnostic and multi‑language.
- **[claude-code v2.1.80 adds rate limit display and inline plugin settings](https://github.com/anthropics/claude-code/releases/tag/v2.1.80)** — Adds a `rate_limits` field to statusline scripts and lets you declare plugins inline in settings.json, improving visibility and extensibility.
- **[oh-my-openagent](https://github.com/code-yeongyu/oh-my-openagent)** — Harness (formerly oh-my-opencode) that standardizes integrating LLM agents into dev workflows, with logging, monitoring, and easy backend swapping.
- **[figma-console-mcp](https://github.com/southleft/figma-console-mcp)** — Bridge exposing a Figma design system as an API, allowing agents to read, create, and debug UI components programmatically.
